    Namespace Morpheo.Core.Sync.Strategies

    Classes

    CompositeRoutingStrategy

    A routing strategy that combines multiple sub-strategies (providers).

    ExternalDbSyncStrategy<TContext>

    Synchronization strategy that pushes changes to an external database (e.g., PostgreSQL, SQL Server).

    HttpPushStrategy

    Strategy that pushes synchronization logs to a central server via HTTP.

    MeshBroadcastStrategy

    Default strategy: Floods the network (Gossip). Ideal for pure Mesh mode without a central server.

    NullRoutingStrategy

    No-op routing strategy.

    NullStrategyProvider

    No-op strategy provider.

    SignalRClientStrategy

    Strategy for connecting to a SignalR Hub as a client. Relays local changes to the hub and receives remote changes.

    SignalRHubConnectionWrapper

    Real implementation wrapping the actual HubConnection.

    SignalRServerBroadcastStrategy

    Strategy for broadcasting changes to all connected SignalR clients (Server-side).

    Interfaces

    IHubConnectionWrapper

    Abstraction for HubConnection to facilitate testing.
